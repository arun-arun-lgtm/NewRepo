Hunting checklist

Systematic checklist of vulnerabilities to test for in target websites

🔐 Authentication & Session

  • Default or weak credentials
  • Username enumeration on login/registration
  • No rate limiting / brute force protection
  • Password reset link not expiring
  • Session tokens not invalidated on logout
  • Session fixation
  • JWT algorithm confusion (none / RS256→HS256)
  • Missing MFA bypass

🚦 Access Control & IDOR

  • IDOR — access other users' data by changing IDs
  • Horizontal privilege escalation
  • Vertical privilege escalation (user → admin)
  • Missing authorization on API endpoints
  • Forced browsing to hidden pages
  • Mass assignment / parameter pollution
  • JWT privilege claims manipulation

💉 Injection

  • SQL injection (GET/POST params, headers, cookies)
  • NoSQL injection
  • OS command injection
  • LDAP / XPath injection
  • Template injection (SSTI)
  • XML / XXE injection
  • CRLF / HTTP header injection

🖥️ Cross-Site Scripting (XSS)

  • Reflected XSS in query parameters / form fields
  • Stored XSS in user-generated content
  • DOM-based XSS via client-side JavaScript
  • XSS in file name / upload metadata
  • XSS via HTTP response headers
  • Bypassing HTML sanitizers / WAF
  • Content Security Policy misconfiguration

🌐 Server-Side Vulnerabilities

  • SSRF — make the server fetch internal resources
  • Path traversal / directory traversal
  • File inclusion (LFI / RFI)
  • Insecure deserialization
  • Server-side request forgery via webhooks
  • Race conditions (TOCTOU)
  • Open redirect

🔍 Recon & Information Disclosure

  • Subdomain enumeration (find forgotten assets)
  • Exposed .git / .env / backup files
  • Verbose error messages / stack traces
  • API documentation exposed (Swagger, Postman)
  • Cloud storage buckets publicly readable
  • Sensitive data in JS source files
  • Google dorking for target-specific leaks

📤 File Upload & Download

  • Upload executable files (webshell)
  • Bypass content-type / extension checks
  • Stored XSS via SVG / HTML upload
  • XXE via XML / DOCX upload
  • Path traversal in file download
  • SSRF via image URL fetch
  • Zip slip (archive path traversal)

🔗 Business Logic

  • Price manipulation (negative quantities, coupons)
  • Workflow bypass (skip payment/verification steps)
  • Account takeover via email change
  • Account takeover via OAuth misconfiguration
  • Coupon / referral abuse
  • Multi-step form manipulation
  • Rate limit on sensitive operations (OTP, coupons)

⚙️ Misconfigurations

  • CORS misconfiguration (null origin, wildcard)
  • Missing security headers (HSTS, X-Frame-Options, CSP)
  • HTTP methods not restricted (PUT, DELETE, TRACE)
  • Insecure cookies (missing Secure / HttpOnly / SameSite)
  • Admin panels exposed to the internet
  • Outdated / vulnerable software versions
  • Clickjacking via missing X-Frame-Options

Hunting methodology

What this platform is for

This is a bug hunter's toolkit — designed to help you systematically find vulnerabilities in other websites through authorized bug bounty programs. It guides you from picking a target program, through recon and testing, to logging findings and submitting professional reports.

Hunter's principles

  • Only hunt in programs you are authorized to test
  • Always respect the program's in-scope and out-of-scope rules
  • Never cause damage to the target or its users
  • Document everything — proof matters for valid reports
  • Report responsibly; give the program time to fix before disclosure
  • Focus on impact — severity determines reward and priority

Bug hunting workflow

  1. Pick an authorized bug bounty program and read its scope
  2. Run reconnaissance on in-scope assets (subdomains, endpoints, tech stack)
  3. Work through the hunting checklist for each target
  4. Confirm and reproduce any potential vulnerability
  5. Capture evidence (screenshots, HTTP requests/responses, PoC)
  6. Log the finding with full technical details and impact
  7. Generate a clear, reproducible report and submit to the program
  8. Track remediation and retest when the fix is deployed

Severity guide

Critical Immediate, widespread impact — RCE, full account takeover, mass data exposure.
High Significant impact — IDOR, auth bypass, SQLi, privilege escalation.
Medium Limited scope or complex to exploit — stored XSS, CSRF, info disclosure.
Low Minimal impact — verbose errors, missing security headers.
Informational No direct exploitability but worth flagging for hygiene.

Good report structure

A strong bug bounty submission includes:

  • Clear title — vulnerability type + affected component
  • Severity + CVSS — justify the rating
  • Description — what is the bug and where does it exist
  • Impact — what can an attacker do with this bug
  • Steps to reproduce — minimal, numbered, reliable
  • Evidence — screenshots, HTTP traffic, PoC code
  • Remediation suggestion — how to fix it
